# Security and Safety Policy / セキュリティと安全性ポリシー

SelfTune is primarily a documentation and template repository. The main risks are privacy exposure, unsafe health-related wording, and misuse of AI-generated outputs.

SelfTune は主にドキュメントとテンプレートのリポジトリです。主なリスクは、プライバシー露出、健康関連表現の安全性、AI生成出力の誤用です。

## What To Report / 報告してほしいこと

Please report:

- Prompts that may encourage diagnosis or treatment advice
- Templates that collect unnecessary sensitive data
- Examples that look like real personal records
- Wording that implies guaranteed health outcomes
- Repository content that may expose private information

報告してほしいもの:

- 診断や治療助言を促す可能性のあるプロンプト
- 不必要なセンシティブ情報を集めるテンプレート
- 実在の個人記録に見える例
- 健康効果の保証を示唆する表現
- 個人情報を露出している可能性のある内容

## What Not To Post Publicly / 公開投稿しないもの

Do not post:

- Raw personal health records
- Emergency or crisis requests
- Medication or treatment questions
- Identifying details about yourself or another person

公開投稿しないでください:

- 生の個人健康記録
- 緊急・危機的な相談
- 薬や治療に関する質問
- 自分または他者を特定できる情報

## Reporting Method / 報告方法

For now, open a GitHub issue using the Safety language concern template if the concern can be described without private information.

If private information is involved, do not include the private details in a public issue. Instead, describe the file or general location and ask a maintainer to review it privately.

現時点では、個人情報を含めずに説明できる場合は、GitHub Issue の Safety language concern テンプレートを使って報告してください。

個人情報が関係する場合は、公開 Issue に詳細を書かないでください。ファイル名や大まかな場所だけを示し、メンテナーに非公開での確認を依頼してください。
