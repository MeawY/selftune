# Contributing / コントリビューション

Thank you for considering a contribution to SelfTune.

SelfTune is a non-medical, open-source toolkit for structured self-observation, reflection, and lifestyle review. Contributions should help people record and organize personal experience without making clinical claims.

SelfTune への貢献を検討してくださりありがとうございます。

SelfTune は、自己観察、内省、生活習慣レビューのための非医療的なオープンソース・ツールキットです。貢献内容は、臨床的な主張ではなく、個人の経験を記録・整理しやすくすることを目的としてください。

## Contribution Types / 貢献の種類

- Improve documentation clarity
- Add safer prompt templates
- Add daily, weekly, or monthly reflection templates
- Add fictional examples
- Improve bilingual wording
- Add privacy or data-handling guidance
- Report unclear, risky, or overly medical language

- ドキュメントの読みやすさの改善
- より安全なプロンプトテンプレートの追加
- 日次、週次、月次の振り返りテンプレートの追加
- 架空のサンプルケースの追加
- 日英表現の改善
- プライバシーやデータ管理に関する指針の追加
- 不明瞭、危険、医療的に見えすぎる表現の報告

## Safety Rules / 安全ルール

Please do not add content that:

- Diagnoses illness or mental health conditions
- Claims to treat, cure, prevent, or reverse a condition
- Gives medication, supplement, or treatment instructions
- Tells users to ignore professional care
- Presents personal tracking data as clinical evidence
- Uses fictional examples that look like real private data

追加しないでください:

- 病気やメンタルヘルス状態を診断する内容
- 治療、治癒、予防、改善を断定する内容
- 薬、サプリメント、治療に関する指示
- 専門家への相談を避けるよう促す内容
- 個人の記録を臨床的証拠として扱う内容
- 実在の個人情報に見えるサンプル

## Writing Style / 文体

- Use plain Markdown.
- Keep sections modular and reusable.
- Prefer checklists, examples, and short paragraphs.
- Use cautious language: "may", "might", "consider", "could be worth tracking".
- Add English and Japanese headings for major sections when practical.

- プレーンな Markdown を使う。
- セクションは小さく、再利用しやすくする。
- チェックリスト、例、短い段落を優先する。
- 「可能性がある」「検討する」「記録するとよさそう」など慎重な表現を使う。
- 主要セクションには可能な範囲で英語と日本語の見出しを付ける。

## Pull Request Checklist / PRチェックリスト

Before opening a pull request, please check:

- The content is non-medical and does not imply diagnosis or treatment.
- Any examples are clearly fictional.
- The language is practical and understandable to non-specialists.
- The change fits the existing folder structure.
- Markdown headings and lists render cleanly.

Pull request を作る前に確認してください:

- 内容が非医療的で、診断や治療を示唆していない。
- 例は明確に架空である。
- 非専門家にも理解しやすい実用的な文体である。
- 既存のフォルダ構成に合っている。
- Markdown の見出しとリストが読みやすく表示される。
