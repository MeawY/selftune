# AGENTS.md

This repository contains SelfTune, an open-source toolkit for AI-assisted self-observation, reflection, and lifestyle review.

## Working Principles

- Develop this repository in English first.
- New durable documentation, templates, prompts, examples, issues, and pull requests should be written in English.
- Do not add bilingual headings by default. Add translations later as separate localized files when needed.
- Keep all content non-medical.
- Do not claim diagnosis, treatment, prevention, cure, or guaranteed health benefits.
- Use cautious phrasing such as "may", "can help organize", "might be worth discussing", and "consider".
- Frame outputs as reflection, journaling, pattern review, and preparation for professional conversations.
- Encourage qualified professional support for severe, persistent, sudden, or concerning symptoms.
- Do not provide medication, supplement, or treatment instructions.
- Prefer clear Markdown with practical examples.
- Keep language plain, practical, and readable by non-specialists.

## Audience

Write for:

- Individuals using AI for structured self-reflection
- Therapists, coaches, and wellness practitioners
- Developers building non-medical personal knowledge tools
- Markdown and Obsidian users

The tone should be practical, calm, and readable by non-specialists while remaining credible for professionals.

## Repository Structure

- `README.md`: project overview and entry point
- `docs/`: conceptual documents and models
- `prompts/`: reusable prompts for AI-assisted review
- `templates/`: reusable Markdown templates
- `examples/`: sample cases and example outputs

## Safety Language

Prefer:

- "This pattern may be worth tracking."
- "Consider discussing this with a qualified professional."
- "This is not a diagnosis."
- "Use this as a reflection aid, not medical advice."

Avoid:

- "This means you have..."
- "This will improve..."
- "You should stop/start medication..."
- "This cures..."
- "This protocol treats..."

## Contribution Style

- Keep files small and modular.
- Prefer checklists and reusable sections.
- Add examples that are clearly fictional.
- Avoid collecting unnecessary personal data.
- When in doubt, make the content safer, more transparent, and easier to adapt.
