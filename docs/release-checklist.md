# Release Checklist / リリースチェックリスト

Use this checklist before publishing or tagging a SelfTune release.

SelfTune を公開またはタグ付けする前に、このチェックリストを使います。

## Documentation / ドキュメント

- [ ] README explains the project purpose clearly.
- [ ] README says SelfTune is not medical advice.
- [ ] `docs/disclaimer.md` is present and linked.
- [ ] `docs/privacy.md` is present and linked.
- [ ] `CONTRIBUTING.md` explains safety rules.
- [ ] `LICENSE` is present.

- [ ] README が目的を明確に説明している。
- [ ] README に SelfTune が医療助言ではないことが書かれている。
- [ ] `docs/disclaimer.md` があり、リンクされている。
- [ ] `docs/privacy.md` があり、リンクされている。
- [ ] `CONTRIBUTING.md` に安全ルールが書かれている。
- [ ] `LICENSE` がある。

## Safety Review / 安全レビュー

- [ ] No file claims diagnosis, treatment, prevention, cure, or guaranteed health effects.
- [ ] No file instructs medication, supplement, or treatment changes.
- [ ] Prompts tell AI systems not to diagnose or provide treatment advice.
- [ ] Examples are fictional or anonymized.
- [ ] Severe, sudden, persistent, worsening, or concerning symptoms are directed toward qualified professional support where relevant.

- [ ] 診断、治療、予防、治癒、健康効果保証を主張しているファイルがない。
- [ ] 薬、サプリメント、治療の変更を指示しているファイルがない。
- [ ] プロンプトがAIに診断や治療助言をしないよう指示している。
- [ ] 例が架空または匿名化されている。
- [ ] 強い症状、急な変化、長引く不調、悪化、不安がある場合に、必要に応じて専門家支援へつないでいる。

## Privacy Review / プライバシーレビュー

- [ ] No real personal records are included.
- [ ] No names, emails, addresses, phone numbers, workplace names, or clinic names are included in examples.
- [ ] Templates do not request unnecessary identifying information.
- [ ] Issue templates warn users not to post private records.

- [ ] 実在の個人記録が含まれていない。
- [ ] 例に氏名、メール、住所、電話番号、勤務先、クリニック名が含まれていない。
- [ ] テンプレートが不要な個人特定情報を求めていない。
- [ ] Issue テンプレートが個人記録を投稿しないよう注意している。

## GitHub Setup / GitHub設定

- [ ] Repository description is concise and non-medical.
- [ ] Topics are added, such as `self-tracking`, `journaling`, `markdown`, `obsidian`, `wellness`, `ai-prompts`.
- [ ] Issues are enabled.
- [ ] Pull request template is enabled.
- [ ] Default branch is protected if maintainers want review before merge.

- [ ] リポジトリ説明文が簡潔で非医療的である。
- [ ] `self-tracking`, `journaling`, `markdown`, `obsidian`, `wellness`, `ai-prompts` などの topics を追加している。
- [ ] Issues が有効である。
- [ ] Pull request テンプレートが有効である。
- [ ] レビュー後にマージしたい場合、デフォルトブランチを保護している。
